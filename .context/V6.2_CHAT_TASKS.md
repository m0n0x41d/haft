# v6.2 Chat View — Structured Agent Output + Conversational Mode

> Problem: prob-20260414-001
> Goal: Desktop task view renders structured agent output and supports follow-up messages.
> Reference: Zenflow UX — collapsible tool calls, thinking panels, markdown text, continuous chat.

## Architecture

### How CLI structured output works

**Claude Code:**
```bash
claude -p "$PROMPT" --output-format stream-json --verbose
```
Emits JSONL to stdout. Key event types:
- `{"type":"system","subtype":"init",...}` — session metadata
- `{"type":"assistant","message":{"content":[{"type":"text","text":"..."},{"type":"tool_use","name":"Bash","input":{...}}]}}` — agent response blocks
- `{"type":"result","subtype":"success","result":"..."}` — final result

Content blocks inside assistant message:
- `{"type":"text","text":"markdown here"}` — render as markdown
- `{"type":"tool_use","id":"...","name":"Bash","input":{"command":"git status"}}` — render as tool call card
- `{"type":"tool_result","tool_use_id":"...","content":"output here"}` — render nested in tool call
- `{"type":"thinking","thinking":"reasoning here"}` — render as collapsible thinking panel

**Codex CLI:**
```bash
codex exec --json "$PROMPT"
```
Emits JSONL to stdout. Different format — needs separate parser.

### What changes

**Go backend (desktop/agents.go):**
1. Spawn CLI with structured output flags
2. Parse JSONL line-by-line
3. Emit typed events to frontend via Wails events (not raw text chunks)
4. Keep process alive after first response for follow-up messages
5. Accept follow-up user messages and pipe to process stdin

**React frontend (desktop/frontend/src/pages/Tasks.tsx):**
1. Replace monolithic output `<pre>` with structured message list
2. Render each block type as distinct component
3. Input field stays active while task is running (not just "completed")
4. Follow-up messages appear as user bubbles in the chat

### Event protocol (Go → Frontend)

```typescript
// Replace single TaskOutputEvent with typed blocks
type ChatBlock =
  | { type: "text"; text: string }
  | { type: "thinking"; summary: string; collapsed: boolean }
  | { type: "tool_use"; id: string; name: string; command: string }
  | { type: "tool_result"; tool_use_id: string; output: string; truncated: boolean }
  | { type: "error"; message: string }
  | { type: "user"; text: string }
  | { type: "status"; message: string }  // "Generating...", "3 tools, 4 thoughts"

type ChatEvent = {
  task_id: string;
  block: ChatBlock;
}
```

## Tasks

### Stream G: Backend — Structured Output Parsing

| # | Task | Package | Acceptance | Size |
|---|------|---------|-----------|------|
| G1 | Add ChatBlock types to Go backend | `desktop/agents.go` | Go struct types matching the ChatBlock TypeScript types above. JSON serializable via Wails. | S |
| G2 | Claude stream-json parser: read JSONL, emit ChatBlock events | `desktop/agents.go` | When agent=claude, spawn with `--output-format stream-json`. Parse each JSONL line into ChatBlock. Emit via `runtime.EventsEmit(ctx, "task:chat", ChatEvent{...})`. Text blocks accumulate and emit on newline. Tool use/result emit immediately. Thinking emits as collapsed. | L |
| G3 | Codex --json parser: read JSONL, emit ChatBlock events | `desktop/agents.go` | When agent=codex, spawn with `--json`. Parse codex JSONL format (different from Claude). Map to same ChatBlock types. | M |
| G4 | Conversational mode: keep process alive, accept follow-up stdin | `desktop/agents.go` | After agent responds, do NOT close the process or mark task completed. Task stays "running". New Wails method `SendTaskMessage(taskID, message string)` writes to process stdin. Task only completes when user explicitly closes or agent exits. | L |
| G5 | Wails binding: SendTaskMessage | `desktop/app.go` | New method `(a *App) SendTaskMessage(taskID string, message string) error`. Writes message + newline to running task's stdin pipe. Returns error if task not running. | S |
