# v6.2 Tasks — Dashboard + Implement + Adopt

> Target: 30 days from v6.1.0.
> Goal: one proved execution loop from desktop. Decision → Implement → Verify → PR.
> Principle: one vertical slice, not a feature wave.
> BDD contract: spec/target-system/EXECUTION_CONTRACT.md

## Stream A: Dashboard Unification

| # | Task | Package | Acceptance | Size |
|---|------|---------|-----------|------|
| A1 | Merge Problems + Decisions pages into unified Dashboard | `desktop/frontend/src/pages/` | Single Dashboard.tsx page showing: active decisions section, governance findings section, recent activity | M |
| A2 | Decision cards show Implement button (active decisions only) | `desktop/frontend/src/pages/Dashboard.tsx` | Implement button visible on active DecisionRecords. Disabled for superseded/deprecated. | S |
| A3 | Governance findings section with Adopt/Waive/Reopen buttons | `desktop/frontend/src/pages/Dashboard.tsx` | Stale and drifted decisions shown with inline action buttons. Buttons call backend methods. | M |
| A4 | Implement guards in UI: G1 blocks, G2/G4 warn, invariant-less warns | `desktop/frontend/src/pages/Dashboard.tsx`, `desktop/app.go` | G1: button disabled with tooltip. G2/G4: confirmation dialog. No invariants: warning dialog. | M |
| A5 | Remove old standalone Problems and Decisions pages from nav | `desktop/frontend/src/App.tsx` | Nav shows Dashboard instead of separate pages. Old pages removed or redirected. | S |

## Stream B: Implement Flow (Go backend)

| # | Task | Package | Acceptance | Size |
|---|------|---------|-----------|------|
| B1 | Implement action: create worktree + feature branch from decision | `desktop/agents.go` | `Implement(decisionRef)` creates worktree at `.haft/worktrees/{branch}`, returns worktree path and branch name | M |
| B2 | Build agent prompt from decision context | `desktop/decision_flow.go` | Prompt includes: invariants, affected_files, rationale from SolutionPortfolio, workflow.md content, governing invariants from knowledge graph | M |
| B3 | Spawn agent in worktree with checkpointed mode | `desktop/agents.go` | Agent runs in worktree directory. Checkpointed = pauses after each tool call for user review. Output streams to task view. | L |
| B4 | Post-run invariant verification | `desktop/decision_flow.go`, `internal/graph/` | After agent task completes: check each decision invariant against worktree state. Return pass/fail per invariant with context. | M |
| B5 | On verification pass: baseline + evidence + status update | `desktop/agents.go`, `internal/artifact/` | Take baseline snapshot of affected_files in worktree. Create CL3 evidence linked to decision. Task status → "Ready for PR". | M |
| B6 | On verification fail: surface broken invariants + user options | `desktop/frontend/src/pages/Tasks.tsx` | Show broken invariants with diff context. Buttons: "Fix and retry" / "Reopen as problem" / "Dismiss invariant". | M |
| B7 | "Create PR" action on passed tasks | `desktop/agents.go` | Generate PR body from decision rationale + invariants + verification result. Push branch. Open PR draft (or copy body to clipboard for manual PR). | M |

## Stream C: Adopt Flow

| # | Task | Package | Acceptance | Size |
|---|------|---------|-----------|------|
| C1 | Adopt action on drift finding: create agent thread with context | `desktop/governance.go`, `desktop/agents.go` | `Adopt(findingRef)` creates task with: full DecisionRecord, drift report (modified/missing files), diffs, invariants, affected modules | M |
| C2 | Adopt action on stale finding: agent thread with evidence history | `desktop/governance.go`, `desktop/agents.go` | `Adopt(findingRef)` for stale: DecisionRecord + evidence timeline + R_eff computation + expired items | M |
| C3 | Resolution actions from Adopt thread | `desktop/frontend/src/pages/Tasks.tsx` | After agent investigation: buttons for Re-baseline / Reopen / Waive / Deprecate / Measure. Each calls corresponding `haft_refresh` or `haft_decision` action. | M |
| C4 | Adopt never auto-resolves — agent presents options, user chooses | `desktop/agents.go` | Agent prompt explicitly says: "Present options. Do not execute resolution without user confirmation." Verified by checking agent prompt content. | S |
| C5 | Resolution recorded as RefreshReport | `desktop/governance.go` | Every Adopt resolution creates RefreshReport artifact linked to the finding and decision. | S |

## Stream D: Integration + Polish

| # | Task | Package | Acceptance | Size |
|---|------|---------|-----------|------|
| D1 | Human confirmation dialog for irreversible actions | `desktop/frontend/src/components/` | Confirmation modal for: Implement, Create PR, Reopen, Supersede, Deprecate. Shows what will happen + affected artifacts. | M |
| D2 | Task view shows execution status ladder | `desktop/frontend/src/pages/Tasks.tsx` | Status progression: Planned → Running → Verifying → Ready for PR / Needs attention. Real-time updates. | M |

## Dependency Graph

```
A1 → A2 → A4                (dashboard → implement button → guards)

B3 → B4 → B6                (agent done → verify → failure UI)

```

## Exit Criteria

v6.2 is done when:
- [ ] Dashboard shows decisions + governance findings in one page
- [ ] Implement creates worktree, runs agent, verifies invariants, takes baseline
- [ ] Failed verification shows broken invariants + resolution options
- [ ] "Create PR" generates body from decision rationale
- [ ] Adopt creates agent thread with full context, user resolves
- [ ] All resolutions recorded as RefreshReport
- [ ] D4 smoke test passes end-to-end
