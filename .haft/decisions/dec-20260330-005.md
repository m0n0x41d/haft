# Tool-layer baseline exposure + pre-measure blocker

## 1. Problem Frame

**Signal:** The agent records haft_decision(measure) without first creating a baseline because the local tool schema hides baseline, prompts omit it, and measure only warns after the fact. This produces weak CL1 evidence and misleading UX.

**Constraints:**
- Do not change artifact-layer fallback semantics unless necessary; manual/CLI users may still need weak CL1 measurement.
- Expose baseline in the same local runtime path the agent uses.
- Prefer blocking agent self-measure without baseline rather than silently downgrading evidence.

**Acceptance:** 1) Local haft_decision tool supports action="baseline". 2) Agent-side measure path refuses to measure decisions with affected_files but no baseline, with actionable guidance. 3) Prompts/docs teach implement→baseline→verify→measure. 4) Relevant tests pass.

## 2. Decision

**Selected:** Tool-layer baseline exposure + pre-measure blocker

The failure is in the agent runtime path: baseline exists in artifact/server layers but the local haft_decision tool hides it and allows measure to proceed. The correct minimal fix is to expose baseline in the local tool schema, block agent-side measure when affected files are unbaselined, and update prompts/skills so the expected ritual is implement → baseline → verify → measure.

**Invariants:**
- artifact.Measure keeps its CL1 fallback semantics for non-agent/manual callers.
- The local agent can call baseline explicitly.
- Measure guidance remains actionable and specific.

**Post-conditions:**
- [ ] Expose baseline action in local tool runtime.
- [ ] Add store-backed baseline precheck before measure.
- [ ] Update prompt/skill wording to reflect the correct FPF ritual.

**Admissibility:**
- NOT: Do not silently auto-baseline during measure.
- NOT: Do not hard-break all non-agent callers at the artifact layer in this change.

## 3. Rationale

**Weakest link:** Other call paths that bypass internal/tools/haft.go may still use warning-only semantics until/unless artifact.Measure is hardened later.

## 4. Consequences

**Affected files:** internal/tools/haft.go, internal/agent/prompt.go, internal/cli/skill/h-reason/SKILL.md
