# Show image attachments in sent chat messages

## Signal

After Ctrl+V attaches an image and the message is sent, the TUI shows no visible indication in chat history that an image was attached. Claude Code-style UX shows attachment pills/placeholders in the sent message block.

## Constraints

- Do not change provider payload semantics.
- Do not regress existing text/file attachment display.
- Keep the fix localized to TUI/view-model rendering.

## Optimization Targets

- Attachment visibility in sent messages
- Low implementation risk
- Consistency with current chip-based input UX

## Observation Indicators

- Input chips exist before send, but user chat history only renders plain text.
- Typed image parts now exist in agent messages, so the TUI can render attachment markers without parsing fake text.

## Acceptance

1) Sent user messages visibly indicate attached images in chat history. 2) Text/file attachments remain visible/understandable. 3) No change to provider request behavior. 4) Existing tests still pass or are updated to cover the new UX.

## Blast Radius

internal/tui rendering for user messages and possibly view-model construction from typed user parts; no provider/protocol changes required.

## Reversibility

High: render-only change.
