# Solutions for: Show image attachments in sent chat messages

Problem: prob-20260330-004

## Variants (3)

### V1. Inline attachment pills above user text

Render sent user messages with one line of attachment pills above the message body, reusing the same chip vocabulary as the input area. Images get an image-specific icon/label; text attachments keep the paperclip style.

**Strengths:**
- Closest to Claude Code screenshots.
- Very localized change in user-message rendering/view model.
- Keeps attachments compact and visible without expanding message height too much.

**Weakest link:** If attachment metadata extraction from saved message parts is incomplete, some sent messages may still render without pills.

**Risks:**
- Needs the user message view model to carry attachment metadata, which is not currently displayed.

### V2. Attachment summary line inside user block

Render a dim summary line like '[Image 1] [file.txt]' or '2 attachments' inside the user block before the text body.

**Strengths:**
- Simplest implementation.
- Works even in narrow terminals.
- Less style complexity.

**Weakest link:** May technically solve visibility while still feeling underdesigned.

**Risks:**
- Less visually distinct than dedicated chips.
- Feels like a fallback UX, not an intentional design.

### V3. Separate attachment sub-items beneath user messages

Split each sent user message into multiple chat items: a user text block plus one attachment item per image/file, potentially selectable and expandable later.

**Strengths:**
- Best long-term foundation for previews, open actions, and multimodal metadata.
- Keeps attachment rendering extensible.

**Weakest link:** Too much structure for a problem that only asks for visible sent-attachment indication.

**Risks:**
- Higher blast radius in chat item building and scrolling behavior.
- Overkill for the reported issue.
